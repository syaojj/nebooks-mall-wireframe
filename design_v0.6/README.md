# NE Books · 자사몰 개편 — design_v0.6

Figma 고해상도 디자인 + Confluence PRD를 반영한 프론트엔드 전 화면 정적 구현입니다.
와이어프레임 트랙(`wireframe/` v1.0~v5.2)과는 별개로, 실제 비주얼 디자인을 그대로 반영하는 새 트랙입니다.

## 구성

```
design_v0.6/
├─ index.html          메인(홈)
├─ list.html            교재목록 (교재구매 ↔ 학습자료 탭)
├─ detail.html           교재상세
├─ cart.html             장바구니
├─ checkout.html         주문/결제
├─ order-complete.html   주문완료
├─ mypage.html           마이페이지 (9개 탭)
├─ cs.html               고객센터 (6개 탭)
├─ css/
│  ├─ style.css          공통 토큰·헤더·GNB·브레드크럼·Footer·버튼/뱃지 (모든 페이지 공용)
│  ├─ home.css / detail.css / cart.css / checkout.css / mypage.css / cs.css   페이지별 스타일
├─ js/
│  ├─ shared.js          공통 헤더 인터랙션(검색·고객센터·햄버거 토스트) — 모든 페이지 공용
│  ├─ app.js(list) / home.js / detail.js / cart.js / checkout.js / order-complete.js / mypage.js / cs.js
└─ assets/               Figma에서 추출한 이미지·아이콘
```

브라우저에서 `index.html`을 열면 바로 확인 가능합니다(별도 빌드 불필요). 모든 페이지는 헤더 로고·GNB·장바구니/마이페이지/고객센터 아이콘·Footer 지사안내 링크로 상호 연결되어 있습니다.

## 백엔드 미구현 영역 (토스트로 안내)

실제 백엔드·PG·SSO 연동이 필요한 동작(결제 실행, NE Tutor/Waffle 이동, 학습자료 실제 다운로드, 통합회원 정보수정, 스윗트래커 배송조회 등)은 토스트 메시지로 대체했습니다. 반면 아래 흐름은 클라이언트 로직으로 실제 동작합니다.

- 교재목록: 탭 전환, 정렬, 필터(레벨/제품유형/브랜드), 판매중만 보기, 찜, 공유 URL 복사
- 교재상세: 탭 스크롤 이동(스크롤스파이), 수량 변경, 찜, 리뷰 작성 폼
- 장바구니: 수량 변경, 선택/삭제, 금액·배송비·적립포인트 실시간 계산, 품절 상품 주문 차단
- 주문/결제: 배송지 탭 전환, 결제수단 선택, 포인트 사용(100P 단위·한도 캡), 순차 필수값 검증, 주문번호 생성 후 `localStorage`로 주문완료 화면에 전달
- 주문완료: 정상 주문 세션이 없으면 UC-ORD-14(비정상 접근 차단)에 따라 차단 화면 노출
- 마이페이지: 9개 탭 전환, 주문상세 인라인 전환, 찜 삭제, 로그아웃
- 고객센터: 6개 탭 전환, 공지/FAQ 필터·아코디언, 1:1문의 회원/비회원 폼 검증(비회원은 개인정보 동의 필수)

## 알려진 제약

- 교재목록/교재상세를 제외한 화면은 Figma 디자인이 없어 design_v0.6 토큰 시스템으로 새로 디자인한 것이며, 추후 해당 화면의 Figma 디자인이 나오면 그에 맞춰 재작업이 필요합니다.
- 모든 페이지의 상품/주문/포인트 데이터는 데모용 목데이터이며 페이지 간 상태가 공유되지 않습니다(주문/결제 → 주문완료만 `localStorage`로 연결).
- Paperlogy / Pretendard 폰트는 jsDelivr CDN(`fonts-archive/Paperlogy`, `orioncactus/pretendard`)을 사용합니다 — 오프라인 환경에서는 시스템 폰트로 대체됩니다.
